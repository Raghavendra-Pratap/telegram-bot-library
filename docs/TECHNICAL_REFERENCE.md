## API Reference

### Dashboard (Flask)

All routes are served by the app in `dashboard.start_dashboard(launcher, host, port)`. No auth layer; dashboard is in-process with launcher.

#### Endpoints

| Method | Endpoint | Purpose | Auth |
|--------|----------|---------|------|
| GET | `/` | Serve dashboard HTML (single page with embedded JS) | No |
| GET | `/api/status` | List all bots with running/stopped status | No |
| POST | `/api/start/<bot_id>` | Start a bot | No |
| POST | `/api/stop/<bot_id>` | Stop a bot | No |
| POST | `/api/restart/<bot_id>` | Restart a bot | No |
| GET | `/api/stats` | Per-bot stats (start/stop/restart counts, uptime, errors) | No |

#### GET /api/status

**Response (200):**
```json
{
  "total_bots": 6,
  "running_count": 2,
  "stopped_count": 4,
  "bots": [
    {
      "id": "tg_download_bot",
      "name": "TG Download Bot",
      "description": "Fast file downloads...",
      "status": "running",
      "pid": 12345,
      "port": 8081,
      "uptime_seconds": 3600.5,
      "start_time": "2026-02-15T12:00:00",
      "last_error": null
    },
    {
      "id": "caption_bot",
      "name": "Caption Bot",
      "description": "...",
      "status": "stopped",
      "pid": null,
      "port": null,
      "uptime_seconds": 0,
      "start_time": null,
      "last_error": null
    }
  ]
}
```

#### POST /api/start/<bot_id>, /api/stop/<bot_id>, /api/restart/<bot_id>

**Success:** `{"success": true}` (200).  
**Error:** `{"success": false, "error": "<message>"}` with 404 (bot not found) or 500 (start failed).

#### GET /api/stats

**Response (200):** Object keyed by bot_id:
```json
{
  "tg_download_bot": {
    "name": "TG Download Bot",
    "start_count": 5,
    "stop_count": 2,
    "restart_count": 1,
    "total_uptime_hours": 12.5,
    "error_count": 0
  }
}
```

### TG Download Bot – File Server (aiohttp)

Internal HTTP server for download links. Routes:

| Method | Path | Purpose |
|--------|------|---------|
| GET | `/download/{token}` | Stream file; 404 if token missing/expired, 410 if expired after check |
| GET | `/health` | Health check |

Token is generated by `FileServer.generate_download_link()`; stored in memory with `expires_at` (FILE_RETENTION_HOURS). No auth beyond token.

## Key Types & Interfaces

### Launcher (bot_launcher.py)

```python
# BotProcess - Running bot subprocess info
# Location: bot_launcher.py
@dataclass
class BotProcess:
    bot_id: str
    bot_name: str
    process: subprocess.Popen
    start_time: datetime
    pid: int
    port: Optional[int] = None
    status: str = "running"
    restart_count: int = 0
    last_error: Optional[str] = None
```

**Stats (per bot_id):** `defaultdict` with:
- `start_count`, `stop_count`, `restart_count`: int
- `total_uptime`: float (seconds)
- `errors`: list (recent error strings)

**Config (bots_config.json) shape:**
- `bots`: list of `{ id, name, description, directory, script, venv_path, port?, enabled? }`
- `dashboard`: `{ port, host }`

### Index Bot – Database Models (SQLAlchemy)

Location: `Index_bot/database.py`

| Model | Table | Purpose |
|-------|-------|---------|
| Channel | channels | Monitored channel (channel_id, channel_username, channel_title, is_active) |
| FileUpload | file_uploads | File in channel (channel_id FK, message_id, file_name, parsed_name, confirmed_name, is_confirmed, needs_confirmation) |
| MovieSeries | movie_series | Confirmed title (name, tmdb_id, media_type) |
| CustomList | custom_lists | Named list of channel IDs (list_name, channel_ids as comma-separated text, is_default) |

DB path: from `Config.DB_PATH` (default `index_bot.db`), SQLite.

### TG Download Bot – User Manager (allowed_users.json)

Location: `TG_download_bot/user_manager.py` (reads/writes `USERS_FILE`, default `allowed_users.json`).

**Schema:**
- `allowed_users`: list of user IDs (int)
- `admin_users`: list of user IDs (int)
- `pending_requests`: object, keys = user_id (str), value = `{ "username", "timestamp", "message_id" }`

### TG Download Bot – FileServer in-memory store

`FileServer.file_store`: dict `token -> { "path", "filename", "expires_at", "created_at" }`. Cleaned on access when expired or file missing.

## State Management

**Approach:** In-process state in launcher; no Redux/global front-end store. Dashboard is server-rendered HTML + JS that fetches `/api/status` and calls POST APIs for actions.

**Store location:** `bot_launcher.py` – `BotLauncher` instance holds:
- `running_bots: Dict[str, BotProcess]`
- `stats: defaultdict` (per-bot counts and errors)
- `config`: loaded from `bots_config.json`
- `monitoring: bool`, `monitor_thread`

**Key mutations:** Start/stop/restart update `running_bots` and `stats`; monitor thread may call `start_bot` on crash. State is not persisted across launcher restarts.

---

## Data Storage

**Launcher:** No persistent storage. Config from `bots_config.json`; stats and running state in memory only.

**Bots:**
- **Index_bot:** SQLite via SQLAlchemy (`DB_PATH`), tables: channels, file_uploads, movie_series, custom_lists.
- **TG_download_bot:** `allowed_users.json` (allowed_users, admin_users, pending_requests).
- **name-bot:** Optional `users.json` (allowed users) if used.
- **TG_download_bot file server:** In-memory token → file path/expiry; files on disk in `DOWNLOAD_DIR`.

### Index Bot – Schema Summary

| Table | Key columns |
|-------|-------------|
| channels | id, channel_id (unique), channel_username, channel_title, added_at, is_active |
| file_uploads | id, channel_id (FK), message_id, file_name, file_size, file_id, uploaded_at, parsed_name, confirmed_name, is_confirmed, needs_confirmation |
| movie_series | id, name (unique), tmdb_id, media_type, created_at |
| custom_lists | id, list_name, created_by, created_at, is_default, channel_ids (text) |

## Build & Deployment

### Build
No compile step. Python 3.8+ and per-directory `pip install -r requirements.txt` (or launcher: `pip install -r launcher_requirements.txt`).

### Run
- Launcher: `python bot_launcher.py` or `./start_launcher.sh` (from repo root).
- Dashboard: Started from launcher menu (option 8); runs in thread, host/port from `bots_config.json`.
- Bots: Started by launcher as subprocesses (`python bot.py` in each bot directory, with that cwd and optional venv).

### Environments
| Environment | Purpose | Config |
|-------------|---------|--------|
| Local | Development / single operator; **dashboard is local-only** (no auth; all bots on your device) | .env per bot; bots_config.json; optional venvs |
| Production | Same codebase; may run launcher on a server | Same; ensure tokens and API keys set; FILE_SERVER_BASE_URL etc. for TG_download_bot if exposed |

No CI/CD or container files observed in repo.

**Dashboard security:** Dashboard has no authentication. Current intent is local-only use (all bots run on the operator’s device). If exposed (e.g. reverse proxy), add auth or restrict access.

**Launcher stats:** In-memory only; lost on restart. Persistence (e.g. logging or JSON) can be planned; not required for current use.

**Index Bot:** Under development; `FileUpload.confirmed_name` ↔ `MovieSeries.name` is by string match; FK or consistency checks can be planned later. See `docs/decisions/DECISIONS.md`.
